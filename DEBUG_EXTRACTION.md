# Debug Output Extraction Guide

## Quick Start

Extract all diagnostic and debug output from a build log with one command:

```bash
./extract-debug-output.sh build.log
```

This creates three files:
- **startup_debug.txt** - Startup file installation tracking from build stages
- **toolchain_diag.txt** - Toolchain configuration and final state verification
- **all_debug.txt** - Combined output from both

## Debug Output Prefixes

All debug output uses consistent, greppable prefixes:

| Prefix | Source | Purpose |
|--------|--------|---------|
| `STARTUP_DEBUG:` | build-newlib.sh, build-gcc-final-gdb.sh | Tracks startup file installation at each build stage |
| `TOOLCHAIN_DIAG:` | diagnose-toolchain.sh | Final toolchain configuration and file verification |

## Manual Extraction

If you need to extract specific sections manually:

```bash
# Startup file debug only
grep "STARTUP_DEBUG:" build.log > startup_debug.txt

# Toolchain diagnostics only
grep "TOOLCHAIN_DIAG:" build.log > toolchain_diag.txt

# Both combined
grep -E "STARTUP_DEBUG:|TOOLCHAIN_DIAG:" build.log > all_debug.txt

# Specific section (e.g., newlib check)
grep "STARTUP_DEBUG:" build.log | grep -A10 "Newlib Installation"
```

## What Each Debug Section Shows

### STARTUP_DEBUG from build-newlib.sh
- Files searched: `*crt*.o` in install directory
- Sample multilib listing: `thumb/v6-m/nofp/`
- **Answers**: Did newlib install crt0.o?

### STARTUP_DEBUG from build-gcc-final-gdb.sh
- Files searched: `crt*.o` in both build and install directories
- Build directory listing: Shows if files were created
- Install directory listing: Shows if files were copied
- **Answers**: Did GCC build and install crti.o, crtn.o, crtbegin.o, crtend.o?

### TOOLCHAIN_DIAG from diagnose-toolchain.sh
- Compiler version and configuration
- Sysroot path verification
- Search directories used by linker
- Multilib configuration (all 39 variants)
- File search results for all critical files
- Actual file listings in key directories
- **Answers**: What is the final state of the installed toolchain?

## Investigation Workflow

1. **Run build** (CI or local)
2. **Extract debug output**: `./extract-debug-output.sh build.log`
3. **Review startup_debug.txt**: Check if files are built/installed
4. **Review toolchain_diag.txt**: Verify final toolchain state
5. **Analyze findings**: Update TOOLCHAIN_LINKING_DEBUG_LOG.md
6. **Apply fix**: Based on evidence, not speculation

## Example: Analyzing Startup Debug Output

```bash
# After extraction, check for patterns
grep "Found:" startup_debug.txt          # What files exist?
grep "No .* found" startup_debug.txt     # What's missing?
grep "Directory not found" startup_debug.txt  # What directories are missing?
```
