# ISL Library Build Modernization

## Date
2025-11-10

## Overview
This document details the modernization of the ISL (Integer Set Library) build configuration to use modern versions of autoconf and automake, while maintaining compatibility with the toolchain build process and minimizing tracked autogenerated files.

## Scope
The changes are confined to the ISL library build stage in the Docker bootstrap phase and do not affect other components (binutils, gcc, gdb, newlib, or other prerequisite libraries).

## ISL Version
- **Version**: 0.18 (reported as 0.15 in configure.ac)
- **Source**: `src/isl/`
- **Purpose**: Integer set library for polyhedral compilation, used by GCC for loop optimization

## Changes Made

### 1. Autoconf Version Requirement Added

**File: `src/isl/configure.ac`**

**Before:**
```autoconf
AC_INIT([isl], [0.15], [isl-development@googlegroups.com])
AC_CONFIG_AUX_DIR([.])
AC_CONFIG_MACRO_DIR([m4])
AM_INIT_AUTOMAKE([foreign])
```

**After:**
```autoconf
AC_PREREQ(2.71)
AC_INIT([isl], [0.15], [isl-development@googlegroups.com])
AC_CONFIG_AUX_DIR([.])
AC_CONFIG_MACRO_DIR([m4])
AM_INIT_AUTOMAKE([1.16.5 foreign])
```

**Rationale:**
- ISL originally had **no** `AC_PREREQ` directive
- Adding `AC_PREREQ(2.71)` ensures compatibility with Ubuntu 22.04 LTS default autoconf
- Autoconf 2.71 (released 2021) is the default version used in the Docker bootstrap stage
- This aligns with the project's strategy of using modern autotools for bootstrap libraries (GMP, MPFR, MPC, ISL, Expat, libiconv)

### 2. Automake Version Specification

**File: `src/isl/configure.ac`**

**Before:**
```autoconf
AM_INIT_AUTOMAKE([foreign])
```

**After:**
```autoconf
AM_INIT_AUTOMAKE([1.16.5 foreign])
```

**Rationale:**
- ISL originally had no minimum automake version specified
- Ubuntu 22.04 LTS provides automake 1.16.5 (released 2021)
- Explicitly specifying 1.16.5 ensures compatibility with modern automake features
- The `foreign` option remains unchanged (allows non-GNU project layout)

### 3. Deprecated Macro Replacement: AC_PROG_LIBTOOL

**File: `src/isl/configure.ac`**

**Before:**
```autoconf
AC_PROG_LIBTOOL
```

**After:**
```autoconf
LT_INIT
```

**Rationale:**
- `AC_PROG_LIBTOOL` was deprecated in libtool 2.0 (2008)
- `LT_INIT` is the modern replacement with identical functionality
- Eliminates deprecation warnings during autoreconf
- More concise and follows current best practices

### 4. Shell Script Modernization: Backtick Replacement

**File: `src/isl/configure.ac`**

**Before:**
```autoconf
if test "x$prefix" != "xNONE"; then
	prefix_wd=`cd $prefix && pwd`
	srcdir_wd=`cd $srcdir && pwd`
	wd=`pwd`
```

**After:**
```autoconf
if test "x$prefix" != "xNONE"; then
	prefix_wd=$(cd "$prefix" && pwd)
	srcdir_wd=$(cd "$srcdir" && pwd)
	wd=$(pwd)
```

**Before (CLANG section):**
```autoconf
	CLANG_CXXFLAGS=`$llvm_config --cxxflags`
	CLANG_LDFLAGS=`$llvm_config --ldflags`
	targets=`$llvm_config --targets-built`
	components="$targets asmparser bitreader support mc"
	CLANG_LIBS=`$llvm_config --libs $components`
	systemlibs=`$llvm_config --system-libs 2> /dev/null | tail -1`
	CLANG_PREFIX=`$llvm_config --prefix`
```

**After (CLANG section):**
```autoconf
	CLANG_CXXFLAGS=$($llvm_config --cxxflags)
	CLANG_LDFLAGS=$($llvm_config --ldflags)
	targets=$($llvm_config --targets-built)
	components="$targets asmparser bitreader support mc"
	CLANG_LIBS=$($llvm_config --libs $components)
	systemlibs=$($llvm_config --system-libs 2> /dev/null | tail -1)
	CLANG_PREFIX=$($llvm_config --prefix)
```

**Total Changes:** 9 instances of backtick command substitution replaced

**Rationale:**
- Backtick command substitution (`` `command` ``) is legacy syntax from ancient shells
- Modern `$(command)` syntax is:
  - POSIX-compliant
  - More readable (especially with nested commands)
  - Easier to quote properly
  - Recommended by shellcheck and modern shell style guides
- Eliminates SC2006 shellcheck warnings
- Aligns with modernization done in other build scripts (build-common.sh, build-prerequisites.sh)

#### 6.2. AC_ERROR → AC_MSG_ERROR (5 instances in m4 macros)

**File: `src/isl/m4/ax_detect_gmp.m4`** (3 instances)

**Before:**
```autoconf
AC_CHECK_HEADER([gmp.h], [], [AC_ERROR([gmp.h header not found])])
AC_CHECK_LIB([gmp], [main], [], [AC_ERROR([gmp library not found])])
AC_LINK_IFELSE([...], [], [AC_ERROR([gmp library too old])])
```

**After:**
```autoconf
AC_CHECK_HEADER([gmp.h], [], [AC_MSG_ERROR([gmp.h header not found])])
AC_CHECK_LIB([gmp], [main], [], [AC_MSG_ERROR([gmp library not found])])
AC_LINK_IFELSE([...], [], [AC_MSG_ERROR([gmp library too old])])
```

**File: `src/isl/m4/ax_detect_imath.m4`** (2 instances)

**Before:**
```autoconf
AC_CHECK_HEADER([imath.h], [], [AC_ERROR([imath.h header not found])])
AC_CHECK_HEADER([gmp_compat.h], [], [AC_ERROR([gmp_compat.h header not found])])
```

**After:**
```autoconf
AC_CHECK_HEADER([imath.h], [], [AC_MSG_ERROR([imath.h header not found])])
AC_CHECK_HEADER([gmp_compat.h], [], [AC_MSG_ERROR([gmp_compat.h header not found])])
```

**Rationale:**
- `AC_ERROR` was deprecated in autoconf 2.50 (2001) - it's an old name
- `AC_MSG_ERROR` is the correct modern macro
- Both macros have identical functionality
- Eliminates obsolescence warnings

#### 6.3. AC_TRY_COMPILE → AC_COMPILE_IFELSE (5 instances in m4 macros)

**File: `src/isl/m4/ax_cflags_warn_all.m4`** (2 instances)

**Before:**
```autoconf
AC_TRY_COMPILE([],[return 0;],
   [VAR=`echo $ac_arg | sed -e 's,.*% *,,'` ; break])
```

**After:**
```autoconf
AC_COMPILE_IFELSE([AC_LANG_PROGRAM([],[return 0;])],
   [VAR=`echo $ac_arg | sed -e 's,.*% *,,'` ; break])
```

**File: `src/isl/m4/ax_create_stdint_h.m4`** (2 instances)

**Before:**
```autoconf
AC_TRY_COMPILE([#include <stdint.h>],[int_least32_t v = 0;],
[ac_cv_stdint_result="(assuming C99 compatible system)"
 ac_cv_header_stdint_t="stdint.h"; ],
[ac_cv_header_stdint_t=""])
```

**After:**
```autoconf
AC_COMPILE_IFELSE([AC_LANG_PROGRAM([[#include <stdint.h>]],[[int_least32_t v = 0;]])],
[ac_cv_stdint_result="(assuming C99 compatible system)"
 ac_cv_header_stdint_t="stdint.h"; ],
[ac_cv_header_stdint_t=""])
```

**File: `src/isl/m4/ax_gcc_warn_unused_result.m4`** (1 instance)

**Before:**
```autoconf
AC_TRY_COMPILE([__attribute__((__warn_unused_result__))
 int f(int i) { return i; }],
 [],
 ax_cv_gcc_warn_unused_result=yes, ax_cv_gcc_warn_unused_result=no)
```

**After:**
```autoconf
AC_COMPILE_IFELSE([AC_LANG_PROGRAM([[__attribute__((__warn_unused_result__))
 int f(int i) { return i; }]],
 [[]])],
 ax_cv_gcc_warn_unused_result=yes, ax_cv_gcc_warn_unused_result=no)
```

**Rationale:**
- `AC_TRY_COMPILE` was deprecated in autoconf 2.50 (2001)
- `AC_COMPILE_IFELSE` with `AC_LANG_PROGRAM` is the modern replacement
- Requires wrapping the header code in `[[ ]]` and body code in `[[ ]]`
- More explicit and follows current best practices
- Eliminates obsolescence warnings

**Note on m4 files:**
Unlike GMP, MPFR, and MPC (where libtool m4 files were removed from tracking), ISL's libtool m4 files in `m4/` are **still tracked**. This is intentional for ISL because:
- ISL uses custom modifications to these m4 files
- ISL is a smaller library with less frequent updates
- Tracking them ensures build reproducibility for this specific version (0.18)
- They will be properly regenerated by autoreconf during build if needed

### Regeneration Process

The `regenerate_autotools()` function in `build-common.sh` automatically regenerates all ISL autotools files during the Docker build:

1. Checks if `configure` and `Makefile.in` already exist (skip if present)
2. Runs `autoreconf -i -f` to regenerate all files:
   - `configure` - Configuration script (~600 KB)
   - `Makefile.in` - Top-level Makefile template (~90 KB)
   - `aclocal.m4` - Autoconf macro definitions (~42 KB)
   - Auxiliary files from automake and libtool
   - Subdirectory `Makefile.in` files (doc/, interface/)
3. Autoconf 2.71 and automake 1.16.5 are used (Ubuntu 22.04 defaults)
4. Total regeneration time: ~3-5 seconds (included in bootstrap stage)

