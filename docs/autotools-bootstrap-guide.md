# Autotools Bootstrap Guide

## Overview

This guide explains how the GNU Tools for STM32 toolchain handles autotools (autoconf, automake, autogen) during the bootstrap stage. The bootstrap process builds prerequisite libraries needed by the GCC toolchain.

## Philosophy: Minimal Tracked Files

The project follows a **minimal tracking** philosophy for autogenerated files:

- **Source files** (configure.ac, Makefile.am) are always tracked in version control
- **Autogenerated files** (configure, Makefile.in, aclocal.m4) are **regenerated** at build time where possible
- Only essential autogenerated files that are difficult to regenerate are tracked in git

This approach:
- Reduces repository size
- Avoids merge conflicts in generated files
- Ensures builds use current autotools versions
- Makes the build process more transparent

## Bootstrap Stage Components

The bootstrap stage (Docker stage 1) builds these prerequisite libraries:

| Library | Purpose | Autotools Usage | Files Tracked |
|---------|---------|-----------------|---------------|
| **zlib** | Compression | Custom configure (not autotools) | 2 (configure, Makefile.in) |
| **GMP** | Arbitrary precision arithmetic | automake + autoconf | 0 (all regenerated) |
| **MPFR** | Multiple precision floating-point | automake + autoconf | 0 (all regenerated) |
| **MPC** | Complex number arithmetic | automake + autoconf | 0 (all regenerated) |
| **ISL** | Integer set library | automake + autoconf | 1 (isl_config.h.in) |
| **Expat** | XML parser for GDB | automake + autoconf | 1 (expat_config.h.in) |
| **libiconv** | Character encoding (Windows) | autoconf only | 8 (Makefile.in files) |

**Total autogenerated files tracked**: 12 files (down from 100+ in older approaches)

## The regenerate_autotools() Function

Located in `build-common.sh`, this function regenerates autotools files before building each library.

### Usage

```bash
regenerate_autotools <library_src_dir>
```

### What It Does

1. **Checks if files already exist**: Skip regeneration if configure and Makefile.in are present
2. **Sets correct autoconf version**: Uses autoconf 2.69 for binutils/gcc/gdb/newlib (required for reproducibility)
3. **Runs autogen if needed**: For projects using Makefile.def (binutils, gcc, gdb, newlib)
4. **Installs auxiliary files**: Copies install-sh, missing, config.guess, config.sub from automake
5. **Runs autoreconf -i -f**: Regenerates configure, Makefile.in, aclocal.m4, config.h.in
6. **Processes subdirectories**: Recursively regenerates nested configure.ac files

### Special Handling

#### Autoconf 2.69 (binutils/gcc/gdb/newlib)

These components require exactly version 2.69 for reproducible builds:

```bash
export AUTOCONF=autoconf2.69
export AUTOHEADER=autoheader2.69
export AUTOM4TE=autom4te2.69
export AUTORECONF=autoreconf2.69
export AUTOUPDATE=autoupdate2.69
```

The Dockerfile is based on Ubuntu 22.04, which ships with autoconf 2.71 by default, so the `autoconf2.69` package must be installed explicitly.

#### Autogen (Makefile.def → Makefile.in)

binutils, gcc, gdb, and newlib use autogen to generate Makefile.in from Makefile.def:

```bash
autogen Makefile.def
```

This must run before autoreconf.

#### libiconv (Custom ACLOCAL_PATH)

libiconv requires special m4 search paths:

```bash
env ACLOCAL_PATH="m4:srcm4:${ACLOCAL_PATH:-}" autoreconf -i -f
```

## Build Process Flow

### 1. Docker Build

The Dockerfile installs autotools:

```dockerfile
RUN apt-get install --yes --no-install-recommends \
    autoconf2.69 \
    automake \
    autogen \
    libtool
```

### 2. Prerequisites Script

`build-prerequisites.sh` regenerates autotools files for each library:

```bash
regenerate_autotools "$SRCDIR/$GMP"
regenerate_autotools "$SRCDIR/$MPFR"
regenerate_autotools "$SRCDIR/$MPC"
regenerate_autotools "$SRCDIR/$ISL"
regenerate_autotools "$SRCDIR/$EXPAT"
regenerate_autotools "$SRCDIR/$LIBICONV"
```

### 3. Configure and Build

After regeneration, standard autotools build commands:

```bash
./configure --prefix=... --disable-shared
make -j$JOBS
make install
```

## Version Control Strategy

### Excluded from Git (.gitignore)

The following autogenerated files are **excluded** from version control:

```gitignore
# Autotools cache
**/autom4te.cache/

# For GMP, MPFR, MPC (fully regenerated)
src/gmp/configure
src/gmp/Makefile.in
src/gmp/aclocal.m4
src/gmp/config.h.in
src/gmp/**/Makefile.in

# Auxiliary build files
src/gmp/compile
src/gmp/config.guess
src/gmp/config.sub
src/gmp/depcomp
src/gmp/install-sh
src/gmp/ltmain.sh
src/gmp/missing
src/gmp/test-driver
src/gmp/ar-lib
```

### Tracked in Git (Minimal)

Only these essential files are tracked:

```
src/isl/isl_config.h.in        # ISL configuration header template
src/expat/expat_config.h.in    # Expat configuration header template
src/libiconv/Makefile.in       # Custom Makefile.in (8 files total)
```

These files are tracked because:
- They contain hand-written customizations
- They're difficult to regenerate without specific configurations
- They rarely change and don't cause merge conflicts

## Troubleshooting

### Issue: "autoconf2.69: command not found"

**Solution**: Install the autoconf2.69 package:

```bash
apt-get install autoconf2.69
```

### Issue: "autogen: command not found"

**Solution**: Install the autogen package:

```bash
apt-get install autogen
```

### Issue: "configure script not found"

**Cause**: Autotools files haven't been regenerated yet.

**Solution**: Run regenerate_autotools before building:

```bash
regenerate_autotools src/gmp
```

### Issue: Autotools drift warnings

**Cause**: Tracked autogenerated files are out of sync with source files.

**Solution**: Regenerate the files:

```bash
cd src/isl
autoreconf -i -f
git add isl_config.h.in
git commit -m "Regenerate isl_config.h.in"
```

## Best Practices

### 1. Never Edit Generated Files

Never manually edit:
- `configure` scripts
- `Makefile.in` files
- `aclocal.m4` files
- `config.h.in` files

Always edit the source files:
- `configure.ac` or `configure.in`
- `Makefile.am`
- `m4/*.m4` macro files

### 2. Use Correct Autotools Versions

- **binutils/gcc/gdb/newlib**: autoconf 2.69 exactly
- **Other libraries**: automake 1.16+, latest autoconf

### 3. Test Regeneration

After modifying source files, test regeneration:

```bash
cd src/gmp
rm -rf autom4te.cache configure Makefile.in aclocal.m4
autoreconf -i -f
./configure && make
```

### 4. Keep .gitignore Updated

When adding new libraries, update .gitignore to exclude their autogenerated files.

## Performance

### Build Time Impact

Regenerating autotools files adds approximately **30-60 seconds** to the bootstrap stage:

| Library | Regeneration Time |
|---------|-------------------|
| GMP | ~15 seconds |
| MPFR | ~10 seconds |
| MPC | ~5 seconds |
| ISL | ~10 seconds |
| Expat | ~5 seconds |
| libiconv | ~10 seconds |

**Total**: ~55 seconds (vs. 2 hour total build time = 0.8% overhead)

This small overhead is acceptable for the benefits of minimal tracked files.

### Caching

The `regenerate_autotools()` function caches results:
- Checks if configure and Makefile.in already exist
- Skips regeneration if files are present
- Useful for iterative development
