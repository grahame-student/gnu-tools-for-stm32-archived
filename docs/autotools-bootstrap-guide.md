# Autotools Bootstrap Guide

## Overview

This guide explains how the GNU Tools for STM32 toolchain handles autotools (autoconf, automake, autogen) during the bootstrap stage. The bootstrap process builds prerequisite libraries needed by the GCC toolchain.

## Philosophy: Minimal Tracked Files

The project follows a **minimal tracking** philosophy for autogenerated files:

- **Source files** (configure.ac, Makefile.am) are always tracked in version control
- **Autogenerated files** (configure, Makefile.in, aclocal.m4) are **regenerated** at build time where possible
- Only essential autogenerated files that are difficult to regenerate are tracked in git

This approach:
- Reduces repository size
- Avoids merge conflicts in generated files
- Ensures builds use current autotools versions
- Makes the build process more transparent

## Bootstrap Stage Components

The bootstrap stage (Docker stage 1) builds these prerequisite libraries using **modern autotools** (autoconf 2.71, automake 1.16.5, libtool 2.4.6):

| Library | Purpose | Autotools Usage | Autotools Versions | Files Tracked |
|---------|---------|-----------------|-------------------|---------------|
| **zlib** | Compression | Custom configure (not autotools) | N/A | 2 (configure, Makefile.in) |
| **GMP** | Arbitrary precision arithmetic | automake + autoconf 2.71 | Modern | 0 (all regenerated) |
| **MPFR** | Multiple precision floating-point | automake + autoconf 2.71 | Modern | 0 (all regenerated) |
| **MPC** | Complex number arithmetic | automake + autoconf 2.71 | Modern | 0 (all regenerated) |
| **ISL** | Integer set library | automake + autoconf 2.71 | Modern | 1 (isl_config.h.in) |
| **Expat** | XML parser for GDB | automake + autoconf 2.71 | Modern | 1 (expat_config.h.in) |
| **libiconv** | Character encoding (Windows) | autoconf 2.71 only | Modern | 8 (Makefile.in files) |

**Note**: Subsequent build stages (binutils, gcc, gdb, newlib) use autoconf 2.69 for reproducible builds as required by those upstream projects.

**Total autogenerated files tracked**: 12 files (down from 100+ in older approaches)

## The regenerate_autotools() Function

Located in `build-common.sh`, this function regenerates autotools files before building each library.

### Usage

```bash
regenerate_autotools <library_src_dir>
```

### What It Does

1. **Checks if files already exist**: Skip regeneration if configure and Makefile.in are present
2. **Sets correct autoconf version**: Uses autoconf 2.69 for binutils/gcc/gdb/newlib (required for reproducibility)
3. **Runs autogen if needed**: For projects using Makefile.def (binutils, gcc, gdb, newlib)
4. **Installs auxiliary files**: Copies install-sh, missing, config.guess, config.sub from automake
5. **Runs autoreconf -i -f**: Regenerates configure, Makefile.in, aclocal.m4, config.h.in
6. **Processes subdirectories**: Recursively regenerates nested configure.ac files

### Special Handling

#### Modern Autotools for Bootstrap Libraries (GMP, MPFR, MPC, ISL, Expat, libiconv)

Bootstrap prerequisite libraries use **modern autotools versions** from Ubuntu 22.04:

- **autoconf 2.71** (default) - Latest stable version with improved features and diagnostics
- **automake 1.16.5** - Current stable release with latest automake best practices
- **libtool 2.4.6** - Modern library building tool
- **autogen 5.18.16** - Current stable release

These modern versions provide:
- Better error messages and diagnostics
- Improved cross-compilation support
- Updated macro definitions
- Enhanced portability checks
- Faster regeneration times

#### Autoconf 2.69 (binutils/gcc/gdb/newlib Only)

Only the main toolchain components (binutils/gcc/gdb/newlib) require the legacy autoconf 2.69 for reproducible builds:

```bash
export AUTOCONF=autoconf2.69
export AUTOHEADER=autoheader2.69
export AUTOM4TE=autom4te2.69
export AUTORECONF=autoreconf2.69
export AUTOUPDATE=autoupdate2.69
```

The Dockerfile installs both autoconf (2.71 default) and autoconf2.69 package. The `regenerate_autotools()` function automatically selects the correct version based on the component being built.

#### Autogen (Makefile.def → Makefile.in)

binutils, gcc, gdb, and newlib use autogen to generate Makefile.in from Makefile.def:

```bash
autogen Makefile.def
```

This must run before autoreconf.

#### libiconv (Custom ACLOCAL_PATH)

libiconv requires special m4 search paths:

```bash
env ACLOCAL_PATH="m4:srcm4:${ACLOCAL_PATH:-}" autoreconf -i -f
```

## Build Process Flow

### 1. Docker Build

The Dockerfile installs modern autotools (Ubuntu 22.04 defaults) plus legacy autoconf 2.69:

```dockerfile
RUN apt-get install --yes --no-install-recommends \
    autoconf \        # 2.71 - Modern version for bootstrap libraries
    autoconf2.69 \    # Legacy version for binutils/gcc/gdb/newlib
    automake \        # 1.16.5 - Latest stable
    autogen \         # 5.18.16 - Latest stable  
    libtool           # 2.4.6 - Modern version
```

**Installed versions (Ubuntu 22.04):**
- autoconf: 2.71 (used for GMP, MPFR, MPC, ISL, Expat, libiconv)
- autoconf2.69: 2.69 (used only for binutils, gcc, gdb, newlib)
- automake: 1.16.5
- libtool: 2.4.6
- autogen: 5.18.16

### 2. Prerequisites Script

`build-prerequisites.sh` regenerates autotools files for each library:

```bash
regenerate_autotools "$SRCDIR/$GMP"
regenerate_autotools "$SRCDIR/$MPFR"
regenerate_autotools "$SRCDIR/$MPC"
regenerate_autotools "$SRCDIR/$ISL"
regenerate_autotools "$SRCDIR/$EXPAT"
regenerate_autotools "$SRCDIR/$LIBICONV"
```

### 3. Configure and Build

After regeneration, standard autotools build commands:

```bash
./configure --prefix=... --disable-shared
make -j$JOBS
make install
```

## Version Control Strategy

### Excluded from Git (.gitignore)

The following autogenerated files are **excluded** from version control:

```gitignore
# Autotools cache
**/autom4te.cache/

# For GMP, MPFR, MPC (fully regenerated)
src/gmp/configure
src/gmp/Makefile.in
src/gmp/aclocal.m4
src/gmp/config.h.in
src/gmp/**/Makefile.in

# Auxiliary build files
src/gmp/compile
src/gmp/config.guess
src/gmp/config.sub
src/gmp/depcomp
src/gmp/install-sh
src/gmp/ltmain.sh
src/gmp/missing
src/gmp/test-driver
src/gmp/ar-lib
```

### Tracked in Git (Minimal)

Only these essential files are tracked:

```
src/isl/isl_config.h.in        # ISL configuration header template
src/expat/expat_config.h.in    # Expat configuration header template
src/libiconv/Makefile.in       # Custom Makefile.in (8 files total)
```

These files are tracked because:
- They contain hand-written customizations
- They're difficult to regenerate without specific configurations
- They rarely change and don't cause merge conflicts

## Troubleshooting

### Issue: "autoconf2.69: command not found"

**Solution**: Install the autoconf2.69 package:

```bash
apt-get install autoconf2.69
```

### Issue: "autogen: command not found"

**Solution**: Install the autogen package:

```bash
apt-get install autogen
```

### Issue: "configure script not found"

**Cause**: Autotools files haven't been regenerated yet.

**Solution**: Run regenerate_autotools before building:

```bash
regenerate_autotools src/gmp
```

### Issue: Autotools drift warnings

**Cause**: Tracked autogenerated files are out of sync with source files.

**Solution**: Regenerate the files:

```bash
cd src/isl
autoreconf -i -f
git add isl_config.h.in
git commit -m "Regenerate isl_config.h.in"
```

## Best Practices

### 1. Never Edit Generated Files

Never manually edit:
- `configure` scripts
- `Makefile.in` files
- `aclocal.m4` files
- `config.h.in` files

Always edit the source files:
- `configure.ac` or `configure.in`
- `Makefile.am`
- `m4/*.m4` macro files

### 2. Use Correct Autotools Versions

- **binutils/gcc/gdb/newlib**: autoconf 2.69 exactly
- **Other libraries**: automake 1.16+, latest autoconf

### 3. Test Regeneration

After modifying source files, test regeneration:

```bash
cd src/gmp
rm -rf autom4te.cache configure Makefile.in aclocal.m4
autoreconf -i -f
./configure && make
```

### 4. Keep .gitignore Updated

When adding new libraries, update .gitignore to exclude their autogenerated files.

## Performance

### Build Time Impact

Regenerating autotools files adds approximately **30-60 seconds** to the bootstrap stage:

| Library | Regeneration Time |
|---------|-------------------|
| GMP | ~15 seconds |
| MPFR | ~10 seconds |
| MPC | ~5 seconds |
| ISL | ~10 seconds |
| Expat | ~5 seconds |
| libiconv | ~10 seconds |

**Total**: ~55 seconds (vs. 2 hour total build time = 0.8% overhead)

This small overhead is acceptable for the benefits of minimal tracked files.

### Caching

The `regenerate_autotools()` function caches results:
- Checks if configure and Makefile.in already exist
- Skips regeneration if files are present
- Useful for iterative development

## Upgrading Autotools Versions

### When to Upgrade

Consider upgrading autotools when:
- Security vulnerabilities are discovered in current versions
- New features in newer versions would benefit the build process
- The Ubuntu base image is upgraded and ships with newer versions
- Upstream projects (GCC, binutils, gdb) update their autotools requirements

### Upgrade Process

#### 1. Check Current Versions

```bash
autoconf --version
automake --version
libtool --version
autogen --version
```

#### 2. Review Compatibility

Before upgrading:
- Check if binutils/gcc/gdb still require exactly autoconf 2.69
  - Review upstream documentation and configure.ac files
  - If they now support newer versions, update the Dockerfile
- Verify prerequisite libraries (GMP, MPFR, MPC, ISL, Expat) work with newer versions
  - Test regeneration with new autotools
  - Check for deprecation warnings

#### 3. Update Dockerfile

If upgrading to newer autotools versions, update the Dockerfile's bootstrap stage:

```dockerfile
# Before (Ubuntu 22.04 with autoconf 2.69)
RUN apt-get install --yes --no-install-recommends \
    autoconf2.69 \
    automake \
    autogen \
    libtool

# After (if binutils/gcc/gdb now support autoconf 2.71+)
RUN apt-get install --yes --no-install-recommends \
    autoconf \
    automake \
    autogen \
    libtool
```

#### 4. Update regenerate_autotools() Function

If autoconf version requirements change, update `build-common.sh`:

```bash
# Current: Forces autoconf 2.69 for binutils/gcc/gdb/newlib
if [ "$lib_name" = "binutils" ] || [ "$lib_name" = "gcc" ] || [ "$lib_name" = "gdb" ] || [ "$lib_name" = "newlib" ]; then
    local autoconf_version=$(autoconf --version 2>/dev/null | head -n1 | grep -oP '\d+\.\d+' || echo "")
    if [ "$autoconf_version" != "2.69" ] && which autoconf2.69 > /dev/null 2>&1; then
        export AUTOCONF=autoconf2.69
        # ... set other variables
    fi
fi

# If upgrading to autoconf 2.71+, update version check:
if [ "$lib_name" = "binutils" ] || [ "$lib_name" = "gcc" ] || [ "$lib_name" = "gdb" ] || [ "$lib_name" = "newlib" ]; then
    local autoconf_version=$(autoconf --version 2>/dev/null | head -n1 | grep -oP '\d+\.\d+' || echo "")
    # Update required version
    if [ "$autoconf_version" != "2.71" ] && which autoconf2.71 > /dev/null 2>&1; then
        export AUTOCONF=autoconf2.71
        # ... set other variables
    fi
fi
```

#### 5. Test Regeneration

After updating autotools versions, test that regeneration works:

```bash
# Test prerequisite library regeneration
cd src/gmp
rm -rf autom4te.cache configure Makefile.in aclocal.m4
autoreconf -i -f
# Check for warnings or errors

# Repeat for MPFR, MPC, ISL, Expat, libiconv
```

#### 6. Validate Full Build

Run a complete Docker build to ensure everything works:

```bash
docker build --target bootstrap -t test-bootstrap .
# Should complete without errors

# Test subsequent stages
docker build --target binutils-gcc-first -t test-stage2 .
docker build --target newlib -t test-stage3 .
# etc.
```

#### 7. Update Documentation

After successful upgrade, update:
- This guide (autotools-bootstrap-guide.md) - version numbers
- Dockerfile comments - autotools version requirements
- automake-modernization-stage-mapping.md - version constraints

### Handling Automake Upgrades

When upgrading automake (e.g., from 1.16 to 1.17):

1. **Check for deprecations**: Run `automake --warnings=all` on each library
2. **Update Makefile.am if needed**: Address any deprecation warnings
3. **Verify aux file compatibility**: Check that install-sh, missing, etc. are compatible
4. **Test across all prerequisite libraries**: Ensure GMP, MPFR, MPC, ISL, Expat all build correctly

### Handling Libtool Upgrades

When upgrading libtool:

1. **Check ltmain.sh compatibility**: Libtool copies ltmain.sh to projects
2. **Verify library building**: Ensure static/shared library building still works
3. **Test on all platforms**: If supporting macOS, test there too

### Common Upgrade Issues

**Issue**: Autoconf 2.71 generates different configure scripts than 2.69

**Solution**: This is expected. Verify the generated configure scripts work correctly. If tracking configure scripts in git (like for gdb/newlib), regenerate and commit them.

**Issue**: New automake version has stricter warnings

**Solution**: Address the warnings by updating Makefile.am files. Common fixes:
- Replace `INCLUDES` with `AM_CPPFLAGS`
- Add `AM_PROG_AR` to configure.ac for library projects
- Update deprecated macro usage

**Issue**: Build breaks with newer autotools

**Solution**: 
1. Check upstream project repositories for compatibility patches
2. Review autotools changelogs for breaking changes
3. Consider staying on current version until upstream catches up

### Ubuntu 24.04 LTS Migration Guide

When upgrading the base image to Ubuntu 24.04:

#### 1. Update Dockerfile Base Image

```dockerfile
# Change from:
FROM ubuntu:22.04 AS bootstrap

# To:
FROM ubuntu:24.04 AS bootstrap
```

#### 2. Expected Changes

**Libtool 2.4.7 (from 2.4.6):**
- Bug fixes and minor improvements
- Better support for recent compiler versions
- No breaking changes for bootstrap libraries
- Test all libraries build correctly: GMP, MPFR, MPC, ISL, Expat

**Package versions remain stable:**
- autoconf 2.71 (unchanged)
- automake 1.16.5 (unchanged)
- autogen 5.18.16 (unchanged)
- autoconf2.69 still available for binutils/gcc/gdb

#### 3. Testing Checklist for 24.04 Migration

```bash
# 1. Update Dockerfile
sed -i 's/FROM ubuntu:22.04/FROM ubuntu:24.04/g' Dockerfile

# 2. Build bootstrap stage
docker build --target bootstrap -t test-24.04-bootstrap .

# 3. Verify all libraries build
docker run -it test-24.04-bootstrap bash -c "
  cd /root/build/gnu-tools-for-stm32 && \
  ls build-native/host-libs/usr/lib/*.a | wc -l
"
# Should show all libraries built

# 4. Test subsequent stages
docker build --target binutils-gcc-first -t test-24.04-stage2 .
docker build --target newlib -t test-24.04-stage3 .

# 5. Compare build artifacts
docker run test-24.04-bootstrap sha256sum /root/build/gnu-tools-for-stm32/build-native/host-libs/usr/lib/libgmp.a
# Compare checksums with 22.04 build
```

#### 4. Potential Issues and Solutions

**Issue**: libtool 2.4.7 changes library naming or flags

**Solution**: 
```bash
# Check libtool changelog
docker run ubuntu:24.04 apt-get changelog libtool | head -50

# Compare library outputs
diff <(docker run ubuntu:22.04-build ls -la /path/to/libs) \
     <(docker run ubuntu:24.04-build ls -la /path/to/libs)
```

**Issue**: autoconf2.69 not found

**Solution**: Ensure universe repository is enabled:
```dockerfile
RUN sed -i 's/# \(.*universe$\)/\1/g' /etc/apt/sources.list && \
    apt-get update
```

#### 5. Future-Proofing Considerations

**When autoconf 2.72+ becomes available:**
- Monitor GCC/binutils/gdb for autoconf 2.72 support
- Test bootstrap libraries with 2.72 first
- Update documentation with 2.72-specific changes
- Consider phasing out autoconf2.69 when toolchain supports it

**When automake 1.17+ is released:**
- Review automake 1.17 changelog for breaking changes
- Test all Makefile.am files with new version
- Update any deprecated automake macros
- Document migration path in this guide

**Build reproducibility:**
- Pin exact package versions if reproducibility is critical
- Consider using buildinfo files to track dependencies
- Document known-good version combinations

### Rollback Procedure

If upgrade causes issues:

```bash
# In Dockerfile, revert to previous versions
RUN apt-get install --yes --no-install-recommends \
    autoconf2.69 \
    automake=1:1.16.1-* \
    autogen \
    libtool

# Or use specific package versions
RUN apt-get install --yes --no-install-recommends \
    autoconf2.69=2.69-11.1 \
    automake=1:1.16.5-1.3 \
    autogen=1:5.18.16-4 \
    libtool=2.4.6-15build2
```
