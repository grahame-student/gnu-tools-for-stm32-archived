# GMP Library Build Modernization

## Date
2025-11-07

## Overview
This document details the modernization of the GMP (GNU Multiple Precision Arithmetic Library) build configuration to use modern versions of autoconf and automake, while maintaining compatibility with the toolchain build process.

## Scope
The changes are confined to the GMP library build stage in the Docker bootstrap phase and do not affect other components (binutils, gcc, gdb, newlib, or other prerequisite libraries).

## Changes Made

### 1. Autoconf Version Update

**File: `src/gmp/configure.ac`**

**Before:**
```autoconf
AC_PREREQ(2.59)
```

**After:**
```autoconf
AC_PREREQ(2.71)
```

**Rationale:**
- GMP originally required autoconf 2.59 (released 2003)
- Ubuntu 22.04 LTS provides autoconf 2.71 (released 2021)
- Autoconf 2.71 is the default version used in the Docker bootstrap stage
- Updating to 2.71 ensures GMP uses modern autoconf features and is compatible with the current build environment
- This aligns with the project's strategy of using modern autotools for bootstrap libraries (GMP, MPFR, MPC, ISL, Expat, libiconv)

### 2. Automake Version Update

**File: `src/gmp/configure.ac`**

**Before:**
```autoconf
AM_INIT_AUTOMAKE([1.8 gnu no-dependencies subdir-objects])
```

**After:**
```autoconf
AM_INIT_AUTOMAKE([1.16.5 gnu no-dependencies subdir-objects])
```

**Rationale:**
- GMP originally required automake 1.8 (released 2004)
- Ubuntu 22.04 LTS provides automake 1.16.5 (released 2021)
- Automake 1.16.5 is the default version used in the Docker bootstrap stage
- Updating to 1.16.5 ensures compatibility with modern automake features
- The options remain unchanged:
  - `gnu`: Follow GNU Coding Standards
  - `no-dependencies`: Disable automatic dependency tracking (GMP preference)
  - `subdir-objects`: Allow source files in subdirectories

### 3. Deprecated Macro Replacement

**File: `src/gmp/configure.ac`**

**Changed:** All 12 instances of `AC_HELP_STRING` to `AS_HELP_STRING`

**Locations:**
- Line 87: `--enable-assert`
- Line 105: `--enable-alloca`
- Line 130: `--enable-cxx`
- Line 139: `--enable-assembly`
- Line 148: `--enable-fft`
- Line 162: `--enable-old-fft-full`
- Line 176: `--enable-nails`
- Line 195: `--enable-profiling`
- Line 229: `--with-readline`
- Line 239: `--enable-fat`
- Line 249: `--enable-minithres`
- Line 259: `--enable-fake-cpuid`

**Rationale:**
- `AC_HELP_STRING` was deprecated in autoconf 2.62 (2008)
- `AS_HELP_STRING` is the modern replacement with identical functionality
- This change eliminates obsolescence warnings during autoreconf
- The replacement is straightforward and maintains the same behavior

### 4. Other Deprecated Macros (Not Changed)

The following deprecated macros remain in `src/gmp/acinclude.m4` (GMP's custom macro file):
- `AC_FD_CC` (multiple instances) - Used in GMP-specific macros
- `AC_PROG_CC_C99` - Used for C99 compiler detection
- `AC_HEADER_STDC` - Used for standard header detection
- `AC_HEADER_TIME` - Used for time header compatibility
- `AC_TYPE_SIGNAL` - Used for signal type detection
- `AC_OUTPUT` with arguments - Legacy syntax

**Rationale for Not Changing:**
- These macros are part of GMP's custom autoconf macros (acinclude.m4), not the main configure.ac
- GMP upstream maintains these macros for broader compatibility with older systems
- Making minimal changes to avoid disrupting GMP's upstream compatibility
- The warnings are informational and do not affect build functionality
- Future GMP upstream versions will address these as part of their own modernization

### Regeneration Process
The `regenerate_autotools()` function in `build-common.sh` automatically regenerates all GMP autotools files during the Docker build:

1. Checks if `configure` and `Makefile.in` already exist (skip if present)
2. Runs `autoreconf -i -f` to regenerate all files:
   - `configure` - Configuration script (955 KB)
   - `Makefile.in` - Top-level Makefile template (57 KB)
   - `aclocal.m4` - Autoconf macro definitions (357 KB)
   - `config.h.in` - Header template for configuration
   - Auxiliary files from automake (install-sh, missing, config.guess, config.sub, etc.)
3. Automake version 1.16.5 and autoconf 2.71 are used (Ubuntu 22.04 defaults)
4. Total regeneration time: ~5-10 seconds (included in bootstrap stage)
