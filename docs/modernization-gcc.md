# GCC Autotools Modernization

## Overview

This document describes the autotools modernization process for the GCC (GNU Compiler Collection) component of the GNU Tools for STM32 toolchain. GCC is built in two stages: GCC First Pass (C compiler only, Stage 2) and GCC Final (C++ compiler with runtime libraries, Stage 4).

## Component Information

- **Component**: GCC (GNU Compiler Collection)
- **Source Directory**: `src/gcc/`
- **Build Stages**: 
  - Stage 2 (binutils-gcc-first): GCC first pass (C compiler only)
  - Stage 4 (gcc-final-gdb): GCC final pass (C and C++ compilers with runtime libraries)
- **Build Scripts**: 
  - `build-binutils-gcc-first.sh` (GCC first pass)
  - `build-gcc-final-gdb.sh` (GCC final pass)
- **Dockerfile Stages**: 
  - `binutils-gcc-first`
  - `gcc-final-gdb`

## Autotools Usage

### Build System Architecture

GCC uses a hybrid autotools approach similar to binutils:

1. **Top Level**: Uses `autogen` to generate `Makefile.in` from `Makefile.def`
   - Does NOT use automake at top level
   - Uses autoconf 2.69 (required for reproducibility)
   - Makefile.def defines the build structure
   - Makefile.tpl provides the template

2. **Subdirectories**: Mixed approach
   - Some subdirectories use automake (e.g., libphobos, libffi, libgfortran)
   - Most tracked subdirectories use hand-crafted Makefile.in files
   - Subdirectories with configure.ac are regenerated during build

### Autotools Versions

- **autoconf**: 2.69 (REQUIRED - exact version for reproducibility)
- **automake**: 1.16+ (Ubuntu 24.04 provides 1.16.5)
- **autogen**: Any version (for Makefile.def processing)
- **libtool**: Any recent version

**Why autoconf 2.69?**
Binutils, GCC, GDB, and Newlib require exactly autoconf 2.69 for reproducible builds. Using autoconf 2.71+ will generate different configure scripts and cause build inconsistencies. The build system automatically detects and uses `autoconf2.69` when available.

## Build Process

### Regeneration During Build

The `regenerate_autotools()` function in `build-common.sh` handles GCC regeneration:

1. **Check for existing files** (lines 274-278):
   - If `configure` and `Makefile.in` exist, skip regeneration
   - Prevents unnecessary rebuilds

2. **Generate Makefile.in from Makefile.def** (lines 310-318):
   ```bash
   if [ -f "Makefile.def" ] && which autogen > /dev/null 2>&1; then
       autogen Makefile.def
   fi
   ```

3. **Use autoconf2.69** (lines 293-306):
   ```bash
   if [ "$lib_name" = "gcc" ]; then
       export AUTOCONF=autoconf2.69
       export AUTOHEADER=autoheader2.69
       export AUTORECONF=autoreconf2.69
       export AUTOUPDATE=autoupdate2.69
       autoreconf_cmd="autoreconf2.69"
   fi
   ```

4. **Install auxiliary files** (lines 343-354):
   - Copies install-sh, missing, config.guess, config.sub, mkinstalldirs, etc.
   - From automake's libdir to GCC root
   - These files are needed for the build system but not tracked in git

5. **Run autoreconf** (lines 366-371):
   ```bash
   autoreconf2.69 -i -f
   ```

6. **Regenerate subdirectories** (lines 374-388):
   - Finds all subdirectories with configure.ac
   - Runs autoreconf on each
   - Generates configure, Makefile.in, aclocal.m4, config.h.in for subdirectories
   - This includes fixincludes, libatomic, libcc1, libffi, libvtv/testsuite/other-tests

### Build Script Flow

From `build-binutils-gcc-first.sh` (GCC First Pass):

```bash
# 1. Regenerate autotools files
regenerate_autotools $SRCDIR/$GCC

# 2. Configure GCC first pass (C compiler only)
$SRCDIR/$GCC/configure \
    --target=$TARGET \
    --prefix=$INSTALLDIR_NATIVE \
    --enable-languages=c \
    --disable-decimal-float \
    --disable-libffi \
    --with-newlib \
    --without-headers \
    ...

# 3. Build and install
make -j$JOBS all-gcc
make install-gcc
```

From `build-gcc-final-gdb.sh` (GCC Final Pass):

```bash
# 1. Regenerate autotools files
regenerate_autotools $SRCDIR/$GCC

# 2. Configure GCC final (C and C++ compiler)
$SRCDIR/$GCC/configure \
    --target=$TARGET \
    --prefix=$INSTALLDIR_NATIVE \
    --enable-languages=c,c++ \
    --with-headers=yes \
    ...

# 3. Build and install
make -j$JOBS
make install
```
