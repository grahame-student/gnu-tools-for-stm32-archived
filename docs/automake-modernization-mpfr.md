# MPFR Library Build Modernization

## Date
2025-11-07

## Overview
This document details the audit and validation of the MPFR (Multiple Precision Floating-Point Reliable Library) build configuration to ensure it uses modern versions of autoconf and automake, while maintaining compatibility with the toolchain build process.

## Scope
The changes are confined to the MPFR library build stage in the Docker bootstrap phase and do not affect other components (binutils, gcc, gdb, newlib, or other prerequisite libraries).

## MPFR Version
- **Version**: 3.1.6
- **Source**: `src/mpfr/`
- **Purpose**: Multiple precision floating-point library for GCC

## Build Environment

### Autotools Versions
- **Autoconf**: 2.71 (Ubuntu 22.04 LTS default)
- **Automake**: 1.16.5 (Ubuntu 22.04 LTS default)
- **Libtool**: 2.4.6 (Ubuntu 22.04 LTS default)

### Build Strategy
MPFR uses the `regenerate_autotools()` function from `build-common.sh` to regenerate all autotools-generated files during the build process. This means:
- No autogenerated files are tracked in git
- Files are regenerated fresh on each build using modern autotools
- Reduces repository size and eliminates drift issues

## Build Process

### 1. Autotools Regeneration

The `regenerate_autotools()` function (in `build-common.sh`) performs the following steps for MPFR:

```bash
# Check if files already exist (skip if present)
if [ -f "$lib_src_dir/configure" ] && find "$lib_src_dir" -maxdepth 1 -name "Makefile.in" -print -quit | grep -q .; then
    echo "Autotools files already exist, skipping regeneration"
    return 0
fi

# Regenerate all autotools files
autoreconf -i -f
```

**Autoreconf Steps:**
1. Run `aclocal` to generate `aclocal.m4` from `m4/` macros
2. Run `libtoolize` to install libtool support files
3. Run `autoconf` to generate `configure` from `configure.ac`
4. Run `autoheader` to generate `config.h.in` (if needed)
5. Run `automake` to generate `Makefile.in` from `Makefile.am`
6. Install auxiliary files (install-sh, missing, config.guess, etc.)

### 2. Build Sequence

MPFR is built in the bootstrap stage of the Docker build:

```bash
# From build-prerequisites.sh (lines 169-183)
echo Task [I-2] /$HOST_NATIVE/mpfr/
rm -rf $BUILDDIR_NATIVE/mpfr && mkdir -p $BUILDDIR_NATIVE/mpfr
pushd $BUILDDIR_NATIVE/mpfr

$SRCDIR/$MPFR/configure --build=$BUILD \
    --host=$HOST_NATIVE \
    --target=$TARGET \
    --prefix=$BUILDDIR_NATIVE/host-libs/usr \
    --disable-shared \
    --with-gmp=$BUILDDIR_NATIVE/host-libs/usr

make -j$JOBS
make install
popd
```

**Dependencies:**
- GMP must be built first (MPFR depends on GMP)
- Build location: `build-native/mpfr/`
- Install location: `build-native/host-libs/usr/`
